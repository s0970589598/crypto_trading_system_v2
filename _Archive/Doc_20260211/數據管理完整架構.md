# 數據管理完整架構 📊

## 🎯 核心模組

### `src/analysis/market_analyzer.py`
**統一的數據管理模組**，所有數據操作都通過這個模組完成。

**核心方法**：
- `load_market_data()` - 載入數據（自動檢查並更新過期數據）
- `_fetch_binance_klines()` - 從 Binance API 獲取數據
- `_update_market_data()` - 更新現有數據到最新
- `_update_market_data_for_timestamp()` - 補齊特定時間點的數據
- `detect_missing_gaps()` - 檢測缺失的K線
- `fill_missing_data()` - 自動填補缺失數據
- `calculate_indicators()` - 計算技術指標
- `analyze_market_at_time()` - 分析特定時間點的市場狀態

## 📥 數據獲取方式（4種）

### 1. 手動下載（CLI 命令行）

#### 完整下載（所有時區）
**文件**：`重新下載市場數據_修正時區.py`

**用途**：
- 首次使用系統
- 數據文件損壞或丟失
- 需要重新下載所有數據

**時區**：1m, 3m, 5m, 15m, 1h, 4h, 1d（7個）

**使用方式**：
```bash
python3 重新下載市場數據_修正時區.py
```

**特點**：
- ✅ 下載最近 90 天數據
- ✅ 自動時區轉換（UTC → UTC+8）
- ✅ 支援多交易對（BTCUSDT, ETHUSDT, SOLUSDT）
- ⏱️ 耗時較長（約 5-10 分鐘）

---

#### 快速下載（關鍵時區）
**文件**：`快速重新下載_關鍵時區.py`

**用途**：
- 快速初始化
- 只需要常用時區
- 節省時間

**時區**：1d, 4h, 1h, 15m（4個）

**使用方式**：
```bash
python3 快速重新下載_關鍵時區.py
```

**特點**：
- ✅ 下載最近 90 天數據
- ✅ 自動時區轉換（UTC → UTC+8）
- ✅ 支援多交易對
- ⚡ 速度快（約 2-3 分鐘）

---

#### 快速更新（增量更新）
**文件**：`快速更新市場數據.py`

**用途**：
- 日常更新
- 只更新增量數據（從最後時間到現在）
- 保持數據最新

**使用方式**：
```bash
python3 快速更新市場數據.py
```

**特點**：
- ✅ 只下載新增的K線
- ✅ 效率高，速度快
- ✅ 適合定時執行（如每天一次）
- ⚡ 速度很快（約 30 秒）

---

#### 檢測並填補缺失
**文件**：`檢測並填補缺失數據.py`

**用途**：
- 檢測數據完整性
- 填補中間缺失的K線
- 修復數據問題

**使用方式**：
```bash
# 只檢測，不填補
python3 檢測並填補缺失數據.py

# 自動填補
python3 檢測並填補缺失數據.py --fetch

# 清理備份文件
python3 檢測並填補缺失數據.py --clean
```

**特點**：
- ✅ 自動檢測時間序列中的缺失
- ✅ 智能分組下載
- ✅ 驗證修復結果
- 🔧 適合定期檢查

### 3. 實盤分析自動更新

**頁面**：`pages/trading/live_market_analysis.py`

**位置**：Web 界面 → 實時市場分析

**自動更新時機**：
1. **載入數據時自動檢查**
   - 調用 `load_market_data()` 時
   - 如果數據過期（>24小時），自動更新
   - 用戶無感知

2. **手動強制更新**
   - 點擊「🔄 更新數據」按鈕
   - 強制更新所有 7 個時區
   - 顯示更新進度

**更新邏輯**：
```python
# 自動更新（載入時）
analyzer = MarketAnalyzer()
df = analyzer.load_market_data('BTCUSDT', '1h')
# 內部會檢查數據是否過期，如果過期自動調用 _update_market_data()

# 手動更新（點擊按鈕）
for interval in ['1d', '4h', '1h', '15m', '5m', '3m', '1m']:
    new_data = analyzer._fetch_binance_klines(symbol, interval, last_time, now)
    # 合併並保存
```

**特點**：
- ✅ 自動檢查過期（24小時）
- ✅ 增量更新（只下載新數據）
- ✅ 手動強制更新選項
- ⚡ 速度快（只更新增量）

### 4. 交易覆盤自動補齊

**頁面**：`pages/review/quality_scoring.py`

**位置**：Web 界面 → 交易覆盤 → 質量評分

**自動補齊時機**：
- 分析過去的交易時
- 如果交易時間點的數據不存在或不完整
- 自動補齊缺失的數據

**補齊邏輯**：
```python
# 分析特定時間點
market_analysis = analyzer.analyze_market_at_time(
    symbol='BTCUSDT',
    timestamp=trade_open_time,  # 交易開倉時間（可能是幾天前）
    intervals=['1m', '3m', '5m', '15m', '1h', '4h', '1d']
)

# 內部邏輯：
# 1. 檢查 trade_open_time 是否在數據範圍內
# 2. 如果不在，調用 _update_market_data_for_timestamp() 補齊
# 3. 如果有缺失，調用 fill_missing_data() 填補
```

**特點**：
- ✅ 完全自動，用戶無感知
- ✅ 智能檢測缺失
- ✅ 自動補齊特定時間點
- ✅ 確保覆盤分析準確
