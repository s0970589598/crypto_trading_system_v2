# 實時價格獲取邏輯說明

## 問題

用戶詢問：「當前價格是用 1 分鐘的價格嗎？」

## 原始邏輯

在修改前，當前價格來自**主時區**（默認 1h），這意味著：
- 價格可能延遲最多 1 小時
- 不夠實時

## 修改後的邏輯

### 價格來源優先順序

系統現在按以下順序尋找最實時的價格：

```
1m > 3m > 5m > 15m > 1h > 4h > 1d
```

**邏輯**：
1. 優先使用最短時區的價格（更實時）
2. 如果該時區數據不存在，則使用下一個時區
3. 確保價格盡可能接近實時

### 代碼實現

```python
# 獲取最實時的價格（優先使用最短時區）
current_price = None
price_source = None

for tf in ['1m', '3m', '5m', '15m', '1h', '4h', '1d']:
    if tf in multi_tf and multi_tf[tf].get('price'):
        current_price = multi_tf[tf].get('price')
        price_source = tf
        break

# 如果沒有找到，使用主分析的價格
if not current_price:
    current_price = analysis.get('price', 0)
    price_source = '主時區'
```

## 用戶選項

### 選項 1：默認模式（推薦）

**不勾選「使用 1m 價格」**

- 分析時區：15m, 1h, 4h, 1d
- 價格來源：15m（最短可用時區）
- 優點：
  - ✅ 不需要下載 1m 數據
  - ✅ 數據量小，速度快
  - ✅ 15m 價格已經足夠實時（最多延遲 15 分鐘）
- 適合：
  - 日常使用
  - 中長期交易
  - 不需要極度實時的場景

### 選項 2：實時模式

**勾選「使用 1m 價格」**

- 分析時區：1m, 15m, 1h, 4h, 1d
- 價格來源：1m（最實時）
- 優點：
  - ✅ 價格最實時（最多延遲 1 分鐘）
  - ✅ 適合短線交易
  - ✅ 更精確的進場時機
- 缺點：
  - ⚠️ 需要下載 1m 數據（數據量大）
  - ⚠️ 分析速度稍慢
- 適合：
  - 短線交易
  - 需要精確進場時機
  - 實時監控

## 數據要求

### 默認模式

需要下載的數據：
```bash
python3 快速重新下載_關鍵時區.py
```

包含：15m, 1h, 4h, 1d

### 實時模式

需要額外下載 1m 數據：
```bash
# 方式 1：完整下載（包含 1m）
python3 重新下載市場數據_修正時區.py

# 方式 2：Web 界面
# 數據管理 → 歷史數據下載 → 選擇 1m 時區
```

## 價格實時性對比

| 模式 | 價格來源 | 最大延遲 | 數據量 | 適合場景 |
|------|---------|---------|--------|---------|
| 默認模式 | 15m | 15 分鐘 | 小 | 日常使用、中長期交易 |
| 實時模式 | 1m | 1 分鐘 | 大 | 短線交易、實時監控 |

## 技術細節

### 為什麼不總是使用 1m？

1. **數據量**
   - 1m 數據量是 15m 的 15 倍
   - 90 天的 1m 數據約 12 萬根 K 線
   - 下載和處理時間較長

2. **實用性**
   - 對於大多數交易者，15m 的實時性已經足夠
   - 1m 價格波動較大，可能造成誤判
   - 中長期交易不需要 1m 級別的精度

3. **性能**
   - 分析 1m 數據需要更多計算資源
   - 可能影響 Web 界面響應速度

### 價格更新頻率

| 時區 | K 線週期 | 價格更新頻率 |
|------|---------|-------------|
| 1m | 1 分鐘 | 每分鐘更新 |
| 15m | 15 分鐘 | 每 15 分鐘更新 |
| 1h | 1 小時 | 每小時更新 |
| 4h | 4 小時 | 每 4 小時更新 |
| 1d | 1 天 | 每天更新 |

### 自動刷新建議

如果使用自動刷新功能：

- **1m 模式**：建議刷新間隔 ≥ 60 秒
- **15m 模式**：建議刷新間隔 ≥ 60 秒
- **原因**：避免過於頻繁的 API 請求

## 使用建議

### 場景 1：日內交易（持倉幾小時）

**推薦**：默認模式（15m）

```
理由：
- 15m 價格已經足夠實時
- 不需要極度精確的進場時機
- 數據量小，分析快速
```

### 場景 2：短線交易（持倉幾分鐘到幾十分鐘）

**推薦**：實時模式（1m）

```
理由：
- 需要精確的進場時機
- 1m 價格更接近實時
- 可以捕捉短期波動
```

### 場景 3：波段交易（持倉幾天）

**推薦**：默認模式（15m）

```
理由：
- 不需要實時價格
- 關注中長期趨勢
- 15m 甚至 1h 都足夠
```

### 場景 4：實時監控

**推薦**：實時模式（1m）+ 自動刷新

```
理由：
- 需要實時監控市場變化
- 1m 價格 + 60 秒刷新
- 可以及時發現交易機會
```

✅ **高級選項**：
