# 重新評分操作指南

## 為什麼需要重新評分？

評分系統已從 v2.0 升級至 v2.1，主要改進：
- ✅ 初始分數從 100 降至 70（更合理）
- ✅ 移除持倉時間加分（持倉時間不代表品質）
- ✅ 減少加分幅度（避免分數膨脹）

**結果**：評分將更準確反映交易品質，分數分布更合理。

---

## 操作步驟

### 方法 1：Web 界面重新評分（推薦）

1. **啟動 Web 界面**
   ```bash
   ./啟動Web界面v2.sh
   ```

2. **進入交易覆盤頁面**
   - 在左側選單點擊「交易覆盤」

3. **選擇要重新評分的交易**
   - 可以選擇單筆或多筆交易
   - 建議選擇所有交易重新評分

4. **點擊「自動評分」按鈕**
   - 系統會自動使用新的評分邏輯
   - 等待評分完成

5. **查看新的評分結果**
   - 檢查評分是否更合理
   - 觀察評分分布

---

### 方法 2：使用清除評分腳本

如果想完全重新開始：

```bash
python 清除全部評分.py
```

**注意**：這會清除所有評分和註記，請謹慎使用！

## 新的評分標準

### 90-100 分（優秀）- 10%
**特徵**：
- 多重信號確認（3+ 個）
- 收益率優秀（≥ 20%）
- 盈虧比優秀（≥ 1.5倍建議值）
- 保守槓桿（≤ 10x）
- 自動平倉

**範例**：
- 順勢交易 + RSI/MACD 確認
- 收益 25%（10x 槓桿）
- 盈虧比 1:3
- 自動止盈

---

### 80-89 分（良好）- 20%
**特徵**：
- 雙重信號確認
- 收益率良好（≥ 10%）
- 盈虧比達標
- 適中槓桿（≤ 20x）
- 自動平倉

**範例**：
- 順勢交易 + RSI 確認
- 收益 15%（15x 槓桿）
- 盈虧比 1:2
- 自動止盈

---

### 70-79 分（及格）- 40%
**特徵**：
- 無明顯優缺點
- 收益率一般（5-10%）
- 盈虧比一般
- 槓桿一般（20-50x）
- 可能手動平倉

**範例**：
- 順勢交易
- 收益 8%（20x 槓桿）
- 盈虧比 1:1.5
- 手動止盈

---

### 60-69 分（需改進）- 20%
**特徵**：
- 有明顯缺點
- 收益率偏低或虧損
- 盈虧比不佳
- 槓桿偏高（> 50x）
- 手動平倉（虧損）

**範例**：
- 逆勢交易
- 虧損 -5%（30x 槓桿）
- 盈虧比 1:0.8
- 手動止損

---

### < 60 分（不及格）- 10%
**特徵**：
- 嚴重問題
- 爆倉或大虧
- 極高槓桿（> 100x）
- 嚴重違反紀律

**範例**：
- 爆倉
- 虧損 -20%（100x 槓桿）
- 沒有止損

## 常見問題

### Q1：為什麼我的分數都降低了？
**A**：這是正常的。舊系統評分偏高，新系統更合理。70 分 = 及格，不是低分。

### Q2：我的獲利交易為什麼只有 70 分？
**A**：獲利不代表優秀。如果只是普通獲利，沒有特別優點，70 分是合理的。

### Q3：如何提高評分？
**A**：
- 多重信號確認（趨勢 + RSI + MACD）
- 提高收益率（≥ 10%）
- 改善盈虧比（≥ 2:1）
- 使用保守槓桿（≤ 10x）
- 設置自動止損/止盈

### Q4：舊的評分會保留嗎？
**A**：重新評分會覆蓋舊評分。如果想保留，請先備份 `data/review_history/quality_scores.json`。

### Q5：可以只重新評分部分交易嗎？
**A**：可以。在 Web 界面中選擇特定交易進行評分。
