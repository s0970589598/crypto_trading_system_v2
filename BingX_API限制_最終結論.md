## 🔴 核心問題

**BingX API 無法批次獲取歷史數據！**

## 📊 測試結果

### 測試方法

請求不同時間範圍的數據，觀察實際返回的時間範圍：

| 請求時間範圍 | 請求 startTime | 實際返回時間 | 結果 |
|-------------|---------------|-------------|------|
| 最近 1 天 | 2026-02-08 15:29 | 2026-02-08 15:30 至 2026-02-09 15:29 | ✅ 正確 |
| 最近 3 天 | 2026-02-06 15:29 | 2026-02-08 15:30 至 2026-02-09 15:29 | ❌ 只返回最近 1 天 |
| 最近 7 天 | 2026-02-02 15:29 | 2026-02-08 15:30 至 2026-02-09 15:29 | ❌ 只返回最近 1 天 |

### 結論

**無論 startTime 設為多早，API 都只返回 endTime 之前的最近 1440 根 K線。**

### 方案 1：使用 Binance API（推薦）

**優點**：
- ✅ 支援批次獲取歷史數據
- ✅ 可以獲取任意時間範圍
- ✅ ETH-USDT 價格與 BingX 基本一致
- ✅ API 限制較寬鬆（單次 1000 根，可分批）

**實施**：

```python
import requests
from datetime import datetime, timedelta

def fetch_binance_klines(symbol, interval, start_time, end_time):
    """從 Binance 獲取歷史數據"""
    url = "https://api.binance.com/api/v3/klines"
    
    all_klines = []
    current_start = start_time
    
    while current_start < end_time:
        params = {
            'symbol': symbol,
            'interval': interval,
            'startTime': int(current_start.timestamp() * 1000),
            'endTime': int(end_time.timestamp() * 1000),
            'limit': 1000
        }
        
        response = requests.get(url, params=params)
        klines = response.json()
        
        if not klines:
            break
        
        all_klines.extend(klines)
        
        # 更新起始時間
        last_time = klines[-1][0]
        current_start = datetime.fromtimestamp(last_time / 1000) + timedelta(seconds=1)
        
        time.sleep(0.5)
    
    return all_klines

# 使用
start = datetime(2026, 1, 21)
end = datetime(2026, 2, 5)
klines = fetch_binance_klines('ETHUSDT', '1m', start, end)
# ✅ 可以獲取完整的 1/21 至 2/5 的數據
```

**注意事項**：
- Binance 價格可能與 BingX 略有差異（通常 < 0.1%）
