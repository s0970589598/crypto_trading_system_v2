# BingX API 歷史數據限制說明

## 📊 測試結果總結

### API 限制

| 項目 | 限制 |
|------|------|
| **單次請求最大數量** | 1440 根 K線 |
| **支援的時間週期** | 1m, 3m, 5m, 15m, 30m, 1h, 2h, 4h, 6h, 12h, 1d, 3d, 1w, 1M |
| **歷史數據範圍** | 無明確限制（測試可獲取 365 天前的數據）|
| **請求頻率限制** | 建議間隔 0.5-1 秒 |

### 各週期數據覆蓋範圍

基於 1440 根 K線的限制：

| 週期 | 1440 根覆蓋時間 | 實際應用 |
|------|----------------|---------|
| **1m** | 24 小時 | 適合當日交易分析 |
| **3m** | 3 天 | 適合短期交易分析 |
| **5m** | 5 天 | 適合週內交易分析 |
| **15m** | 15 天 | 適合雙週交易分析 |
| **1h** | 60 天 | 適合月度交易分析 |
| **4h** | 240 天 | 適合季度交易分析 |
| **1d** | 1440 天（約 4 年）| 適合長期交易分析 |

## 🔧 獲取歷史數據的方法

### 方法 1：分批獲取（推薦）

```python
def fetch_bingx_klines(symbol, interval, start_time, end_time):
    """
    分批獲取歷史數據
    """
    all_klines = []
    current_start = start_time
    
    while current_start < end_time:
        params = {
            'symbol': symbol,
            'interval': interval,
            'startTime': int(current_start.timestamp() * 1000),
            'endTime': int(end_time.timestamp() * 1000),
            'limit': 1440  # 最大限制
        }
        
        response = requests.get(url, params=params)
        klines = response.json()['data']
        
        if not klines:
            break
        
        all_klines.extend(klines)
        
        # 更新起始時間為最後一根 K線的時間 + 1 秒
        last_time = klines[-1]['time']
        current_start = datetime.fromtimestamp(last_time / 1000) + timedelta(seconds=1)
        
        # 避免 API 限流
        time.sleep(0.5)
    
    return all_klines
```

## ⚠️ 注意事項

1. **API 限流**
   - 請求間隔至少 0.5 秒
   - 避免短時間內大量請求
   - 如果被限流，等待 1 分鐘後重試

2. **數據準確性**
   - BingX 數據可能與實際交易時略有差異
   - 建議以實際交易記錄為準
   - 技術指標僅供參考

3. **時區問題**
   - BingX API 返回的時間戳是 UTC
   - 需要轉換為你的本地時區
   - 腳本已自動處理

4. **數據存儲**
   - 原始數據會保存到本地
   - 避免重複獲取
   - 定期清理舊數據
