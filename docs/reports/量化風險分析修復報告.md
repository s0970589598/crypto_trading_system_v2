## 問題描述

### 錯誤訊息
```
❌ 量化風險分析失敗：'total_loss_in_streak'
```

### 錯誤位置
- **檔案**: `pages/review/quality_scoring.py`
- **行數**: 1765
- **代碼**:
```python
st.metric(
    "最長連損",

    delta_color="inverse"

```

---

## 根本原因

### 問題 1: 空數據返回不完整
**位置**: `src/analysis/quantitative_risk.py` (第 770 行)

**原代碼**:
```python
def calculate_max_losing_streak(self) -> Dict:
    if self.df is None or len(self.df) == 0:
        return {'max_streak': 0, 'details': []}  # ← 缺少 total_loss_in_streak
```

**問題**: 當沒有交易數據時，返回的字典缺少 `total_loss_in_streak` 欄位。

---

### 問題 2: 未傳入交易數據
**位置**: `pages/review/quality_scoring.py` (第 1750 行)

**原代碼**:
```python
# 創建風險分析器實例
risk_analyzer = QuantitativeRiskAnalyzer()  # ← 沒有傳入數據
```

**問題**: `QuantitativeRiskAnalyzer` 被創建時沒有傳入交易數據，導致 `self.df = None`。

---

## 修復方案

### 修復 1: 完善空數據返回值
**檔案**: `src/analysis/quantitative_risk.py`

**修改前**:
```python
if self.df is None or len(self.df) == 0:
    return {'max_streak': 0, 'details': []}
```

**修改後**:
```python
if self.df is None or len(self.df) == 0:
    return {'max_streak': 0, 'total_loss_in_streak': 0.0, 'details': []}
```

**改善**: 確保返回值包含所有必要欄位，即使沒有數據。

---

### 修復 2: 正確傳入交易數據
**檔案**: `pages/review/quality_scoring.py`

**修改前**:
```python
from src.analysis.quantitative_risk import QuantitativeRiskAnalyzer

# 創建風險分析器實例
risk_analyzer = QuantitativeRiskAnalyzer()
```

**修改後**:
```python
from src.analysis.quantitative_risk import QuantitativeRiskAnalyzer

# 準備交易數據（需要轉換為 QuantitativeRiskAnalyzer 期望的格式）
trades_for_risk = scores_df.copy()

# 確保必要的欄位存在
if 'pnl' not in trades_for_risk.columns and 'profit_loss' in trades_for_risk.columns:
    trades_for_risk['pnl'] = trades_for_risk['profit_loss']

if 'close_time' not in trades_for_risk.columns and 'exit_time' in trades_for_risk.columns:
    trades_for_risk['close_time'] = trades_for_risk['exit_time']

# 添加 is_loss 欄位
if 'is_loss' not in trades_for_risk.columns:
    trades_for_risk['is_loss'] = trades_for_risk['pnl'] < 0

# 創建風險分析器實例並傳入數據
risk_analyzer = QuantitativeRiskAnalyzer(trades_df=trades_for_risk)
```

**改善**: 
1. 正確傳入交易數據
2. 處理欄位名稱差異（`profit_loss` vs `pnl`）
3. 添加必要的 `is_loss` 欄位

```python
def __init__(self, trades_df: Optional[pd.DataFrame] = None):
    if trades_df is not None:
        required_columns = ['pnl', 'close_time']
        missing = [col for col in required_columns if col not in trades_df.columns]
        if missing:
            raise ValueError(f"Missing required columns: {missing}")
    self.df = trades_df
```
