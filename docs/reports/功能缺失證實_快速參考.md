## 核心發現

### ❌ 完全缺失的演算法

1. **Kelly Criterion 最優倉位計算**
   - 公式: `f* = (bp - q) / b`
   - 破產風險: `RoR = (q/p)^(C/A)`
   - 搜索結果: `grep -r "kelly" src/` → **無匹配**

2. **Tilt Score 傾斜行為評分**
   - 檢測邏輯: 虧損後槓桿/倉位增加 > 20%
   - 評分系統: severity (none/low/medium/high)
   - 搜索結果: `grep -r "tilt" src/` → **無匹配**

3. **情緒失控係數 (Emotional Control Score)**
   - 評分範圍: 0-100
   - 檢測指標: 虧損後下單頻率增加、槓桿增加
   - 搜索結果: `grep -r "emotional_control" src/` → **無匹配**

4. **能力維度評分 (Skill Dimension Scoring)**
   - 五大維度: 方向研判力、風險控管力、心理韌性、執行紀律、成本意識
   - 評分範圍: 0-10 (每個維度)
   - 搜索結果: `grep -r "skill_dimension" src/` → **無匹配**

5. **槓桿壓力測試 (1x-20x)**
   - 系統性測試: 1x, 2x, 3x, 5x, 10x, 20x
   - 爆倉檢測: `bankrupted = final_capital <= 0`
   - 搜索結果: `grep -r "leverage.*comparison" src/` → **無匹配**

6. **失控交易檢測**
   - 定義: 虧損後 + (間隔<30分鐘 或 持倉<5分鐘 或 槓桿>80x)
   - 統計對比: 失控交易 vs 正常交易
   - 搜索結果: `grep -r "is_tilt" src/` → **無匹配**

## 關鍵差異

### 被動式 vs 主動式

**現行系統（被動式）**:
- 設定閾值（如最大回撤 15%）
- 超過閾值則暫停交易
- 事後分析虧損原因

**缺失功能（主動式）**:
- 計算最優倉位（Kelly Criterion）
- 評估破產風險（Risk of Ruin）
- 實時檢測情緒失控
- 預測行為模式（Tilt Score）

### 事後分析 vs 行為檢測

**現行系統（事後分析）**:
- 分析單筆交易虧損原因
- 分類：止損太緊、趨勢判斷錯誤等

**缺失功能（行為檢測）**:
- 檢測交易行為模式
- 識別：虧損後報復性交易、情緒失控
