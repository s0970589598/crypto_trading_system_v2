## 概述

本指南幫助您從舊的 `quantitative_risk_analysis.py` 遷移到新的模組化架構。

### 主要變更

1. **模組化設計**: 從單一檔案拆分為多個模組
2. **核心整合**: 整合到 RiskManager, PerformanceMonitor, LossAnalyzer
3. **CLI 工具**: 提供命令行接口
4. **向後兼容**: 保留兼容層，舊代碼無需修改

## 從舊版本遷移

### 選項 1: 使用兼容層（推薦）

**優點**: 無需修改現有代碼

**步驟**:
1. 保持現有代碼不變
2. 系統會自動使用新實現
3. 會顯示棄用警告

**示例**:
```python
# 舊代碼（仍然可用）
from quantitative_risk_analysis import QuantitativeRiskOfficer

risk_officer = QuantitativeRiskOfficer('trades.json')
result = risk_officer.calculate_ror_kelly()

# ⚠️ 會顯示棄用警告，但功能正常
```

---

### 選項 2: 遷移到新 API（建議）

**優點**: 使用最新功能，無棄用警告

**步驟**:
1. 更新導入語句
2. 更新類別名稱（可選）
3. 測試功能

**遷移對照表**:

| 舊代碼 | 新代碼 |
|--------|--------|
| `from quantitative_risk_analysis import QuantitativeRiskOfficer` | `from src.analysis.quantitative_risk import QuantitativeRiskAnalyzer` |
| `risk_officer = QuantitativeRiskOfficer()` | `risk_analyzer = QuantitativeRiskAnalyzer()` |

**示例**:
```python
# 舊代碼
from quantitative_risk_analysis import QuantitativeRiskOfficer
risk_officer = QuantitativeRiskOfficer('trades.json')
result = risk_officer.calculate_ror_kelly()

# 新代碼
from src.analysis.quantitative_risk import QuantitativeRiskAnalyzer
risk_analyzer = QuantitativeRiskAnalyzer('trades.json')
result = risk_analyzer.calculate_ror_kelly()
```

---

### 選項 3: 使用核心模組整合

**優點**: 與系統其他模組無縫整合

**步驟**:
1. 在核心模組中啟用量化分析
2. 調用量化分析方法

**示例**:

#### RiskManager
```python
from src.managers.risk_manager import RiskManager
from src.models.risk import RiskConfig

config = RiskConfig()
rm = RiskManager(config, 10000.0)

# 啟用量化分析
rm.enable_quantitative_analysis('trades.json')

# 使用量化分析功能
kelly = rm.calculate_kelly_criterion()
tilt = rm.detect_tilt_behavior()
```

#### PerformanceMonitor
```python
from src.analysis.performance_monitor import PerformanceMonitor

pm = PerformanceMonitor()

# 啟用量化分析
pm.enable_quantitative_analysis('trades.json')

# 使用量化分析功能
recovery = pm.calculate_recovery_factor()
emotional = pm.analyze_emotional_control()
```

#### LossAnalyzer
```python
from src.analysis.loss_analyzer import LossAnalyzer

la = LossAnalyzer()

# 啟用量化分析
la.enable_quantitative_analysis('trades.json')

# 使用量化分析功能
streak = la.calculate_max_losing_streak()
short_trades = la.analyze_short_term_trades(5.0)
```

## 故障排除

### 問題 1: 導入錯誤

**錯誤信息**:
```
ModuleNotFoundError: No module named 'src.analysis.quantitative_risk'
```

**解決方案**:
1. 確認您在專案根目錄
2. 確認 `src/analysis/quantitative_risk.py` 存在
3. 確認 Python 路徑正確

---

### 問題 2: 數據載入失敗

**錯誤信息**:
```
ValueError: 載入數據失敗：Expecting value: line 1 column 1 (char 0)
```

**解決方案**:
1. 確認數據文件是 JSON 格式（不是 CSV）
2. 確認 JSON 格式正確
3. 確認文件路徑正確

**數據格式示例**:
```json
[
  {
    "trade_id": "T0001",
    "open_time": "2024-01-01 00:00:00",
    "close_time": "2024-01-01 01:00:00",
    "pnl": 50.0,
    "leverage": 5,
    "quantity": 1.0,
    "fee": 0.5
  }
]
```

---

### 問題 3: 方法返回 None

**原因**: 量化分析未啟用

**解決方案**:
```python
# 在核心模組中，需要先啟用
rm.enable_quantitative_analysis('trades.json')

# 然後才能使用
result = rm.calculate_kelly_criterion()
```

---

### 問題 4: 棄用警告太多

**解決方案**:
```python
# 方案 1: 遷移到新 API（推薦）
from src.analysis.quantitative_risk import QuantitativeRiskAnalyzer

# 方案 2: 忽略警告
import warnings
warnings.filterwarnings('ignore', category=DeprecationWarning)
```

---

### 問題 5: 測試失敗

**解決方案**:
```bash
# 運行所有測試
python3 -m pytest tests/ -v

# 只運行量化風險測試
python3 -m pytest tests/analysis/test_quantitative_risk.py -v

# 只運行兼容層測試
python3 -m pytest tests/test_compatibility_layer.py -v
```
