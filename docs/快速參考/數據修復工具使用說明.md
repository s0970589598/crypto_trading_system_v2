# 數據修復工具使用說明

## 🎯 快速使用

### 檢測數據缺失

```bash
python3 檢測並填補缺失數據.py
```

### 自動修復缺失

```bash
python3 檢測並填補缺失數據.py --fetch
```

### 清理舊備份

```bash
python3 檢測並填補缺失數據.py --clean
```

## 📋 功能說明

### 1. 檢測模式（默認）

**命令**：
```bash
python3 檢測並填補缺失數據.py
```

**功能**：
- ✅ 掃描所有 `market_data_*.csv` 文件
- ✅ 檢測缺失的K線
- ✅ 顯示缺失的時間戳
- ✅ 統計缺失數量
- ❌ 不修改任何文件

**輸出示例**：
```
⚠️ 發現 7 個缺失的K線

缺失的時間戳：
   1. 2026-02-09 23:00:00
   2. 2026-02-10 00:00:00
   3. 2026-02-10 01:00:00
   ...
```

---

### 2. 自動修復模式

**命令**：
```bash
python3 檢測並填補缺失數據.py --fetch
# 或
python3 檢測並填補缺失數據.py -f
```

**功能**：
- ✅ 檢測缺失的K線
- ✅ 從 Binance API 獲取缺失數據
- ✅ 合併到原文件
- ✅ 驗證修復結果
- ✅ **修復成功後自動刪除備份** ✨

**流程**：
```
1. 檢測缺失 → 發現 7 個缺失
2. 分組獲取 → 分為 1 個時間段
3. API獲取 → 獲取 8 條數據
4. 合併保存 → 總行數：2179
5. 驗證結果 → ✅ 所有缺失已填補
6. 刪除備份 → ✅ 備份文件已刪除
```

**注意**：
- 需要網絡連接
- 可能需要幾分鐘
- API有請求限制（腳本會自動延遲）

---

### 3. 清理備份模式

**命令**：
```bash
python3 檢測並填補缺失數據.py --clean
# 或
python3 檢測並填補缺失數據.py -c
```

**功能**：
- ✅ 查找所有備份文件
- ✅ 顯示備份文件列表
- ✅ 確認後刪除

**備份文件類型**：
- `market_data_*.csv.backup` - 驗證腳本的備份
- `market_data_*.csv.before_fill` - 修復前的備份

**輸出示例**：
```
找到 5 個備份文件：
   - market_data_BTCUSDT_1h.csv.before_fill
   - market_data_BTCUSDT_4h.csv.before_fill
   ...

確定要刪除這些文件嗎？(y/N): y
   ✅ 已刪除：market_data_BTCUSDT_1h.csv.before_fill
   ...
✅ 共刪除 5 個備份文件
```

## ⚠️ 注意事項

### 1. 備份文件

- ✅ 修復成功 → 自動刪除備份
- ⚠️ 修復失敗 → 保留備份（可手動恢復）
- ⚠️ 修復部分成功 → 保留備份

### 2. API限制

- Binance API 有請求頻率限制
- 腳本會自動延遲（0.5秒/請求）
- 大量缺失可能需要幾分鐘

### 3. 數據安全

- 修復前會先備份原文件
- 只有驗證成功才刪除備份
- 可以隨時用 `--clean` 清理舊備份

### 4. 無法獲取的數據

某些時間段可能真的沒有數據：
```
⚠️ 仍有 1 個缺失（可能是API無數據）
```
這種情況是正常的，備份會保留。

## 🎓 常見問題

### Q1: 為什麼會有缺失數據？

**A:** 可能的原因：
- 網絡中斷導致下載不完整
- API返回數據不完整
- 某些時段市場停盤

### Q2: 修復失敗怎麼辦？

**A:** 備份文件會保留，可以手動恢復：
```bash
mv market_data_BTCUSDT_1h.csv.before_fill market_data_BTCUSDT_1h.csv
```

### Q3: 可以只修復特定文件嗎？

**A:** 目前腳本會掃描所有文件。如需只修復特定文件，可以：
1. 臨時移動其他文件到別的目錄
2. 運行修復
3. 移回文件

### Q4: 備份文件佔用空間怎麼辦？

**A:** 使用清理命令：
```bash
python3 檢測並填補缺失數據.py --clean
```

### Q5: 修復後還是有缺失？

**A:** 可能是：
- API無該時段數據（正常）
- 網絡問題（重試）
- API限制（等待後重試）
