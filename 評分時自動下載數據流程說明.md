# 評分時自動下載數據流程說明

## 問題確認

**問：如果在評分時，沒有數據，也是用這套 UTC+8 時區的邏輯嗎？**

**答：是的！** 所有通過 MarketAnalyzer 下載的數據都會使用 UTC+8 時區。

## 完整流程

## 三種情況都使用 UTC+8

### 情況 1：數據文件已存在（舊數據）

- **問題**：舊文件可能是 UTC 時間
- **解決**：需要重新下載（使用 `快速重新下載_關鍵時區.py`）
- **結果**：新文件使用 UTC+8 時區 ✅

### 情況 2：數據文件不存在（首次下載）

- **流程**：
  1. `load_market_data()` 檢測文件不存在
  2. 自動調用 `_fetch_binance_klines()` 下載
  3. 下載時自動轉換為 UTC+8
  4. 保存到文件
- **結果**：新文件使用 UTC+8 時區 ✅

### 情況 3：數據需要更新（補齊數據）

- **流程**：
  1. `load_market_data()` 檢測數據過期或範圍不足
  2. 調用 `_update_market_data()` 或 `_update_market_data_for_timestamp()`
  3. 這些函數內部調用 `_fetch_binance_klines()` 下載新數據
  4. 新數據自動轉換為 UTC+8
  5. 合併並保存
- **結果**：更新後的文件使用 UTC+8 時區 ✅

## 關鍵代碼位置

所有數據下載都經過同一個函數，確保時區一致：

```
src/analysis/market_analyzer.py
├── load_market_data()              # 載入數據（不存在則下載）
├── _update_market_data()           # 更新過期數據
├── _update_market_data_for_timestamp()  # 補齊特定時間點數據
└── _fetch_binance_klines()         # ⭐ 核心下載函數（UTC+8 轉換）
    └── 第 262-263 行：時區轉換邏輯
```

## 總結

✅ **所有情況都使用 UTC+8 時區邏輯**

無論是：
- 首次下載
- 自動補齊
- 手動更新

所有通過 `_fetch_binance_klines()` 下載的數據都會自動轉換為本地時間（UTC+8），確保與交易記錄的時間一致。

## 注意事項

1. **舊數據文件**：在修復前下載的文件仍是 UTC 時間，需要重新下載
2. **備份**：舊文件已備份到 `market_data_backup_utc/` 目錄
3. **驗證**：可以通過檢查最後一筆數據的時間與當前時間的差異來驗證時區是否正確
   - 正確：時間差 < 2 小時
   - 錯誤：時間差約 8 小時（UTC 時間）
