## 📊 系統運作流程

1. **每 5 分鐘檢查一次市場**
   - 獲取 1D/4H/1H/15M K 線數據
   - 計算 EMA、ATR、RSI 指標

2. **檢查 13 個進場條件**
   - 週期共振（1D/4H/1H 一致）
   - RSI 在 40-60 範圍
   - 價格接近 EMA 20/50
   - 成交量放大
   - 黃金時段/黃金日
   - 交易次數限制
   - 連續虧損檢查
   - 等等...

3. **當所有條件符合時**
   - 發送 Telegram 通知
   - 提供具體進場價、止損價、目標價
   - 計算盈虧比

4. **您決定是否交易**
   - 系統只提醒，不自動交易
   - 最終決策由您做出
   - 建議再次確認圖表

## 🎯 使用建議

### 第一週：觀察模式

1. 啟動系統，但不實際交易
2. 記錄每次提醒
3. 事後驗證準確性
4. 統計勝率

### 第二週：模擬盤測試

1. 在 BingX 模擬盤執行
2. 嚴格按照提醒交易
3. 記錄每筆交易結果
4. 如果勝率 > 40%，進入下一階段

### 第三週：小倉位實盤

1. 槓桿降至 5x
2. 每筆風險 1%（降低一半）
3. 每日最多 1 筆交易
4. 持續記錄覆盤

### 第四週：正常倉位

1. 如果前三週表現良好
2. 恢復正常倉位（2% 風險）
3. 每日最多 2 筆交易
4. 持續優化

## 🚨 重要提醒

### 1. 系統不是聖杯

- 提醒系統基於技術分析
- 無法預測突發事件
- 勝率目標 45-50%（不是 100%）

### 2. 嚴格執行止損

- 系統提供的止損價必須執行
- 絕不移動止損
- 觸及止損立即平倉

### 3. 控制情緒

- 虧損後不要報復性交易
- 連續虧損 3 次休息 1 小時
- 單日虧損 > 5 USDT 停止交易

### 4. 持續學習

- 記錄每筆交易
- 每週覆盤
- 分析錯誤
- 不斷改進

## 🆘 故障排除

### 問題 1：沒收到 Telegram 通知

**檢查**：
```bash
python3 test_telegram.py
```

**可能原因**：
- Bot Token 錯誤
- Chat ID 錯誤
- 網絡連接問題

### 問題 2：系統報錯

**檢查依賴**：
```bash
pip3 install requests pandas numpy
```

### 問題 3：數據獲取失敗

**可能原因**：
- Binance API 限制
- 網絡連接問題

**解決**：
- 增加檢查間隔（300 → 600 秒）
- 使用 VPN
