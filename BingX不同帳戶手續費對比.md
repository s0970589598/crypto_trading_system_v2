# BingX 不同帳戶手續費對比

## ✅ 官方確認

根據 [BingX 官方文檔](https://bingxservice.zendesk.com/hc/en-001/articles/11264421893007)，**不同帳戶類型的手續費確實不同**！

## 📊 手續費對比表

| 項目 | 永續合約（Perpetual Futures） | 標準合約（Standard Futures） |
|------|----------------------------|---------------------------|
| **帳戶類型** | USDT-M Perpetual Futures | Standard Futures |
| **Maker 費率** | 0.020% | - |
| **Taker 費率** | 0.050% | - |
| **統一費率** | - | 0.045% |
| **收費時機** | 開倉 + 平倉 | **僅平倉時收費** ⭐ |
| **VIP 優惠** | ✅ 最低 Maker 0% / Taker 0.028% | ❌ 無 |

## 🔍 關鍵差異

### 1. 永續合約（USDT-M Perpetual Futures）

**費率結構**：Maker-Taker 模型
- **Maker（限價單）**：0.020%
- **Taker（市價單）**：0.050%

**收費時機**：
- ✅ 開倉時收費
- ✅ 平倉時收費
- 總手續費 = 開倉手續費 + 平倉手續費

**計算公式**：
```
開倉手續費 = 倉位價值 × 費率
平倉手續費 = 倉位價值 × 費率
總手續費 = 開倉手續費 + 平倉手續費
```

**例子**：
- 倉位價值：1,000 USDT
- 使用限價單（Maker 0.02%）
- 開倉手續費：1,000 × 0.02% = 0.20 USDT
- 平倉手續費：1,000 × 0.02% = 0.20 USDT
- 總手續費：0.40 USDT

### 2. 標準合約（Standard Futures）

**費率結構**：統一費率
- **統一費率**：0.045%（不區分 Maker/Taker）

**收費時機**：⭐ **重點**
- ❌ 開倉時不收費
- ✅ **僅平倉時收費**

**計算公式**：
```
手續費 = 倉位價值 × 0.045%（僅平倉時）
```

**例子**：
- 倉位價值：1,000 USDT
- 開倉手續費：0 USDT
- 平倉手續費：1,000 × 0.045% = 0.45 USDT
- 總手續費：0.45 USDT

### 分析

#### Standard_Futures（標準合約）

你的費率：**0.0225%**

**計算驗證**：
```
總倉位價值：190,593.70 USDT
如果僅平倉收費 0.045%：
預期手續費 = 190,593.70 × 0.045% = 85.77 USDT ✅

實際手續費：85.77 USDT
實際費率：85.77 / 190,593.70 = 0.045% ✅
```

**但為什麼顯示 0.0225%？**

因為我的計算方式是：
```
費率 = 手續費 / (倉位價值 × 2)
     = 85.77 / (190,593.70 × 2)
     = 85.77 / 381,187.40
     = 0.0225%
```

我錯誤地假設開倉和平倉都收費，所以除以 2 倍交易量。

**正確的計算應該是**：
```
費率 = 手續費 / 倉位價值
     = 85.77 / 190,593.70
     = 0.045% ✅
```

#### USDT_Perpetual（永續合約）

你的數據顯示：
- 訂單數：128 筆
- 手續費：0.00 USDT
- 倉位價值：0.00 USDT

**可能原因**：
1. 數據記錄方式不同（手續費可能記錄在其他欄位）
2. 手續費已包含在盈虧中
3. 使用了手續費返還活動
4. 保證金欄位為空或為 0
