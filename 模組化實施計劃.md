# Web Dashboard 模組化實施計劃

## 🎯 目標

將 5,853 行的單一文件逐步模組化，最終實現：
- 主文件只負責路由（~200 行）
- 每個功能獨立模組（200-500 行）
- 新功能可以直接創建新模組
- 避免所有代碼塞在同一頁

### 階段 1：拆分交易覆盤（優先，2-3 小時）

**步驟**：
1. ✅ 創建目錄結構

3. ✅ 拆分交易記錄管理（~500 行）

5. ✅ 拆分虧損分析（~200 行）
6. ✅ 測試所有功能
7. ✅ 更新主文件路由

**預期結果**：
- 主文件減少到 ~1,100 行
- 交易覆盤變成 4 個獨立模組
- 功能完全正常

### 階段 2：拆分其他大模組（按需，1-2 天）

**候選模組**（按優先級）：
1. 回測系統（327 行）- 可拆分
2. 實盤交易（205 行）- 可拆分
3. 策略管理（141 行）- 可拆分
4. 參數優化（134 行）- 可保留

**拆分原則**：
- 超過 200 行的考慮拆分
- 功能複雜的優先拆分
- 經常修改的優先拆分

**實施方式**：
```bash
# 每次拆分一個模組
1. 創建 pages/module_name.py
2. 將代碼移到新文件
3. 在主文件中導入
4. 測試功能
5. 提交代碼
```

## 💡 關鍵問題解答

### Q1: 拆分後會影響現有功能嗎？
**A**: 不會。拆分只是重新組織代碼，功能完全一樣。

### Q2: 以後新增功能還會塞在同一頁嗎？
**A**: 不會。新功能直接創建新模組文件，主文件只需加一行導入。

### Q3: 其他模組可以隨時拆分嗎？
**A**: 可以。拆分是漸進式的，隨時可以拆分任何模組。

### Q4: 拆分後性能會變差嗎？
**A**: 不會。反而可能更快，因為可以按需載入。

### Q5: 團隊協作會更容易嗎？
**A**: 是的。不同人可以開發不同模組，減少衝突。
